## Game Status Logic

Games transition through three states based on game time:

1. **Scheduled**: Game hasn't started yet (`gameTime > currentTime`)
2. **Live**: Game is in progress (`currentTime >= gameTime && currentTime < gameTime + 3 hours`)
3. **Final**: Game has ended (`currentTime >= gameTime + 3 hours`)

## Score Generation

Scores are deterministic based on game ID:

```typescript
// Example: game-2026-01-25-LAL-BOS
const seed = gameId
  .split('')
  .reduce((acc, char) => acc + char.charCodeAt(0), 0);
const baseScore = 95 + (seed % 30); // Range: 95-124
const scoreDiff = (seed * 7) % 20; // Difference: 0-19
```

This ensures:

- Same game ID always generates same scores
- Realistic NBA score ranges (95-124 points)
- Varied outcomes (close games and blowouts)
- Predictable for testing

## Testing Tips

### Consistent Data

The same date always generates the same games:

```bash
# These will always return the same games
curl 'http://localhost:3000/api/games?date=2026-01-25'
curl 'http://localhost:3000/api/games?date=2026-01-25'
```

### Date Ranges

Generate historical data for analysis:

```typescript
import { generateHistoricalGames } from '@/lib/mockData';

const startDate = new Date('2026-01-01');
const endDate = new Date('2026-01-31');
const allGames = generateHistoricalGames(startDate, endDate);
// Returns all games for January 2026
```

### Custom Fixtures

Use predefined fixtures for specific tests:

```typescript
import { RIVALRY_GAME_FINAL, OVERTIME_GAME } from '@/lib/mockFixtures';

// Test with known data
expect(RIVALRY_GAME_FINAL.homeTeam.name).toBe('Boston Celtics');
expect(OVERTIME_GAME.scores.quarter).toBe('OT');
```

## Integration Path

When ready to switch from mock to real data:

1. **Keep mock functions** for development and testing
2. **Add environment variable** to toggle mock vs. real data:
   ```typescript
   const USE_MOCK_DATA = process.env.NEXT_PUBLIC_USE_MOCK_DATA === 'true';
   ```
3. **Update API routes** to check the flag:
   ```typescript
   if (USE_MOCK_DATA) {
     return generateMockGames(date);
   } else {
     return fetchRealGames(date);
   }
   ```
4. **Maintain mock fixtures** for E2E tests
